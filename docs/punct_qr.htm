PUNCTUATORS

{...}   string brackets (braces)
The string brackets are used to mark text strings as variables when declaring a string in the "variables" period; the symbol used to identify the string will represent the pointer to that string inside the workspace. Remember to use the underscore besides any effective blank spaces inside a text string.


[...]   square brackets
Square brackets are used to reference the CONTENT of memory locations, like variables or locations pointed by registers. If omitted, registers' values or symbols' effective addresses are referenced instead; this mechanism needs you to be careful using brackets where appropriate, and only where appropriate.


(...)   parenthesis
Parenthesis are used to mark comments. Comments may be placed anywhere and have any size: they can contain any characters (except a closing parenthesis of course, which would terminate the comment). Comments may be nidified: this means comments inside other comments are allowed, and ultimately, that you may write comments in common language without having to worry about special characters that may interrupt the comment block; the only thing you have to remember is to always close parenthesis.


_   underscore
The underscore means a blank space, and should be used ONLY within text strings: it causes commands to be misinterpreted. "_" is here because any "real" blank space is always ignored, along with any other control codes below ASCII code 32 (this includes tabulations, carriage return, line feed and suchs).


\   backslash
Used in declared text strings, marks special text format control codes: \cr means a carriage return, \lf means a line feed, \ta means a tabulation, \us means an underscore, and \cs means a closing brace. To mark an effective backslash, type a backslash which is not immediately followed by cr, lf, ta, us or cs, or type a double backslash: "\\".


;   semicolon
Marks the end of an instruction, declaration or directive. Must be always included, unless you're inside a comment. Also period names and code labels don't need the terminal semicolon, since they're already enclosed between quotes.


"   doublequote
Quotes are used to mark both "code labels" and "periods". Code labels are, for example, the destionation of a jump, or the beginning of a subroutine (giving the routine's name). Periods are the units in which the structure of an application is divided; there are complessively 7 periods: "libraries", "stockfile", "directors", "constants", "variables", "workspace" and the "programme" period, and they must be declared one after the other in exactly that order, even if some periods may be omitted when there's no need for them. Code labels, instead, can only be placed in the "programme" period, which contains the executable code forming the main program and all of its subroutines.


***   triple asterisk
The triple asterisk specifies duplicates for variable's value, used when the symbol is a vector rather than a variable. This can be used only within the variables area.
